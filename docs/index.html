In my doctoral work, I make simulations of proteins doing chemical reactions. This involves studying how atoms inside a protein move, particularly when a reaction is happening. For example, take a look at the following figure:

This is a dynamic process of atoms moving in a molecule!

My research involves trying to understand why they move this way, and by using ML techniques to explore this. However, before I can do that, I need to be able to extract useful information out of static, xyz coordinates. For every model I create, there are xyz-coordinates of every atom, for every time-step within a simulation. I need to take these coordinates, and convert them into a text file for another program. The following JupyterNotebook demonstrates how I built a class to do the following:

  1. Extract positions of atoms (340 total from the xyz coordinates to 32 atoms of interest) from binary files
  2. Compute geometric features off the positions of these atoms
  3. Use the position of these atoms to write an input file for another program that will help me extract features of interest.

Note, in order to even build my ML methods, I required many steps of processing on very low-level binary files. This is one of the ways I automated, and streamlined this process.

In the following block, I will load the necessary python packages I use to process (read/write) a variety of different files, some of which were made from FORTRAN, custom-software like CHARMM, MATLAB, text files, and require compatibility between them..

For visualization purposes, show how we've modeled the coordinates of the "reaction-coordinates". This is considering the distance between C4-C5 and C5-C7, the former a bonded pair of atoms and the latter a broken pair of atoms. What you expect to see is a series of trajectories that oscillate (like a sine-wave), stretch quite far after t = 0, and then return back to what would be a typical oscillation.
